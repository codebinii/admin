# Sesión 2026-03-12

## Resumen

Sesión enfocada en trazabilidad y observabilidad de la aplicación. Se implementó un sistema completo de logging por archivos (sin base de datos) y se resolvió un bug crítico de CORS en producción.

---

## Features implementadas

### 1. Sistema de trazabilidad por archivos (`feat: da35bff`)

Logging detallado a disco con rotación diaria y retención configurable. Sin dependencia de base de datos.

**Canales creados:**

| Canal | Archivo | Contenido |
|---|---|---|
| `requests` | `storage/logs/requests/laravel-YYYY-MM-DD.log` | Trazabilidad HTTP in/out |
| `auth_events` | `storage/logs/auth/laravel-YYYY-MM-DD.log` | Eventos de autenticación |
| `app_events` | `storage/logs/app/laravel-YYYY-MM-DD.log` | Eventos de negocio (uso libre) |

**Componentes:**

- `app/Support/AppLog.php` — Helper con `::request()`, `::auth()`, `::event()` y `::mask()`
- `app/Http/Middleware/RequestTracingMiddleware.php` — Intercepta cada request/response, genera `X-Request-ID` (UUID), loguea método, ruta, IP, user_agent, user_id, input mascarado, status y duración en ms
- `app/Services/Auth/AuthService.php` — Loguea: `auth.login.success`, `auth.login.failed`, `auth.register`, `auth.logout`, `auth.logout_all`, `auth.password.changed`

**Control desde `.env`:**

```env
LOG_TRACE_ENABLED=true   # apaga/enciende todo el sistema
LOG_TRACE_DAYS=30        # retención en días
```

**Uso en services:**
```php
AppLog::event('modulo.operacion', ['campo' => $valor]);
```

### 2. Fix CORS producción (`fix: f7c76ae`)

**Causa raíz:** `config/cors.php` no existía. `HandleCors` (middleware global de Laravel) no tenía configuración y rechazaba preflights `OPTIONS` del browser con `405 Method Not Allowed`. En local no se manifestaba porque Postman/CLI no envían preflights.

**Solución:** Se publicó `config/cors.php` con `CORS_ALLOWED_ORIGINS` configurable desde `.env`.

```env
CORS_ALLOWED_ORIGINS=*                                        # desarrollo
CORS_ALLOWED_ORIGINS=https://app.com,https://admin.app.com   # producción
```

Se expone `X-Request-ID` en `exposed_headers` para correlación end-to-end.
